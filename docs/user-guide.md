# 打卡契约（ClockInPact）用户使用指南

打卡契约适合 2-6 人的小团队做长期目标挑战。  
你可以把它理解成“有规则的打卡群”：挑战者负责执行，监督者负责审核，系统负责统计进度和结算结果。

## 先跑通一次（3-5 分钟）

目标：先完成一次“建组并把朋友拉进来”。

1. 注册并登录账号  
你需要填写用户名、密码、昵称。注册成功后会直接进入“我的小组”页。

2. 创建小组  
在“我的小组”页点击“创建小组”，填写小组名称，选择你的角色（挑战者/监督者），再确认小组时区。

3. 复制邀请码发给朋友  
进入新建的小组页后，你会看到 5 个一次性邀请码。点击“复制”，发到你们的聊天群。

4. 让朋友用邀请码加入  
朋友在“加入小组”页输入邀请码并选择角色后，就能加入。

成功标准：你在“成员列表”里看到自己和朋友，且角色显示正确。

## 跟着一个真实场景走完核心流程

### 场景起点

你和 3 位朋友准备做“跑步挑战”：2 位挑战者负责跑步，2 位监督者负责审核证据。

### 第一步：把小组结构搭好（角色要配平）

1. 至少保证 1 位挑战者 + 1 位监督者。  
2. 新成员加入时，按职责选角色：
“挑战者”会参与打卡；
“监督者”只审核，不打卡。
3. 小组内所有日期都按“小组时区”计算，创建后不能改。

判断是否正确：
- 成员列表里角色分配清晰
- 大家认可时区（避免跨时区误会“今天/明天”）

### 第二步：创建目标并让全员确认

在小组页点击“创建目标”，依次填写：
- 目标名称、目标类别
- 达标条件（数值 + 单位）
- 开始日期和结束日期
- 奖惩规则
- 证据要求

你会看到这些关键限制：
- 开始日期必须是未来日期（按小组时区）
- 同一时间小组只能有一个“待确认/待开始/进行中”的目标
- 目标周期受“该类别已完成次数”影响（可创建的最长周期会提示在页面上）

提交后目标进入“待确认”。  
每位成员都要在目标详情页点“同意”或“拒绝”。

判断是否成功：
- 状态从“待确认”变为“待开始”
- 到开始日期当天自动变成“进行中”

### 第三步：挑战者每日打卡，监督者做审核

挑战者操作：
1. 进入目标详情，点击“打卡”
2. 选择打卡日期（可补卡）
3. 填数值，上传 1-5 张证据图（每张不超过 5MB）
4. 提交

监督者操作：
1. 在目标详情的“打卡记录”里查看证据
2. 点“确认”或“质疑”（质疑必须写理由）

关键规则：
- 打卡提交后不能修改、不能删除
- 任一监督者“质疑”，该条记录就标记为“质疑”，不计入累计
- 全体监督者都确认后，记录才“已确认”
- 超过 3 天无人审核，会“自动通过”

判断是否成功：
- 打卡状态出现“已确认”或“自动通过”
- 累计进度开始上涨

### 第四步：看懂进度页与动态流

在“查看进度”里你会看到：
- 我的进度：累计完成、百分比、剩余数值、剩余天数
- 排行榜：按完成百分比排名（可超过 100%）
- 贡献图：按天看状态（已确认/待审核/质疑）

在“小组动态”里你会看到：
- 谁创建目标、谁打卡、谁审核、谁确认结算
- 系统自动动作（如自动通过、状态变更、结算完成、解锁更长周期）

使用建议：
- 每天看一次“待审核/质疑数量”
- 争议优先处理，避免拖到结算阶段

### 第五步：结算、归档、进入下一轮

到结束日期后，目标进入“待结算”。

结算顺序：
1. 先把所有“待审核”记录处理完
2. 全体监督者在“结算确认”页完成确认
3. 系统给出每位挑战者的达标结果并归档

你会看到：
- 每位挑战者的完成值和完成百分比
- 达标/未达标标记
- 奖惩规则回显
- 达标者在该类别解锁更长周期（用于下次创建更长目标）

## 用熟之后：提高效率与避免踩坑

### 常见误区

| 误区 | 后果 | 正确做法 |
|---|---|---|
| 小组角色全是挑战者或全是监督者 | 目标无法进入有效执行阶段 | 建组时先配好至少 1 挑战者 + 1 监督者 |
| 开始日期选成今天 | 创建失败 | 开始日期至少选明天（按小组时区） |
| 先拖着不审核，等结算再说 | 结算卡住 | 每天清一次待审核 |
| 打卡想“先随便交，后面改” | 无法编辑或删除 | 提交前检查日期、数值和证据 |
| 发起修改/取消后不跟进 | 请求可能被拒绝或过期 | 发起后提醒成员尽快投票 |

### 失败恢复（按问题快速处理）

1. 邀请码提示无效  
先确认是否输错，再确认该码是否已被使用；如已用，向组内成员要新的可用邀请码。

2. 打卡被质疑  
先在目标详情查看质疑理由，和监督者对齐证据标准后重新提交一条新打卡。

3. 无法创建新目标  
通常是当前仍有“待确认/待开始/进行中”目标。先处理当前目标状态，再开新目标。

4. 修改/取消请求一直没结果  
查看倒计时和投票状态，尽快让未投票成员处理；过期后可按新方案重新发起。

### 常见问题

**Q1：一个人能加入多个小组吗？**  
可以。

**Q2：监督者能打卡吗？**  
不能。监督者负责审核。

**Q3：挑战者能审核自己的打卡吗？**  
不能，审核由监督者完成。

**Q4：补卡可以补多久以前？**  
可补目标开始日期到今天之间的日期（按小组时区）。

**Q5：为什么我打卡后进度没立刻上涨？**  
因为“待审核/质疑”不计入累计，需变成“已确认”或“自动通过”。

**Q6：超过目标值后会怎样？**  
可以继续累计，百分比会超过 100%。

**Q7：为什么看不到站内评论区？**  
动态流是“只读记录流”，不支持评论和点赞。

**Q8：结算后还能回看历史吗？**  
可以。在小组的历史目标中可继续查看记录和图表。

## 一图总览（全文收束）

你可以把这款应用当成一条固定节奏：
先建组配角色 -> 创建并确认目标 -> 挑战者打卡、监督者审核 -> 看进度和动态 -> 到期结算归档 -> 用解锁周期进入下一轮。
