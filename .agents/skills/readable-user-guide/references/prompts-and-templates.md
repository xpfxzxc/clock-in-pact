## 1) 最终用户指南骨架

```markdown
# <应用名> 使用指南

<1 段概要：这款应用帮谁解决什么问题。>

## 先跑通一次（3-5 分钟）

<给出最短操作路径与成功判定标准。>

## 跟着一个真实场景走完核心流程

### 场景起点
<用户当前处境与目标>

### 第一步：<动作>
<怎么做 + 做完后应看到什么>

### 第二步：<动作>
<怎么做 + 常见卡点>

### 第三步：<动作>
<怎么做 + 成功判定>

## 用熟之后：提高效率与避免踩坑

### 常见误区
<误区 -> 后果 -> 正确做法>

### 失败恢复
<出现异常时先做什么，再做什么>

### 常见问题
<问答式，优先真实高频问题>

## 一图总览（全文收束）

<用一段文字串起“目标-路径-关键节点-结果”。>

![总览图占位](<待补链接>)

## 附：素材与演示

- 配图：<链接或待补说明>
- 视频演示：<链接或待补说明>
```

## 2) AI 配图提示词模板

用于用户通过“手绘信息图助手”生成配图。
不要写风格描述，专注内容信息与结构。

```markdown
【配图目标】
用于解释：<具体章节/具体问题>
读者看完应理解：<一句话结果>

【画面结构（16:9）】
- 左侧：<模块/流程起点>
- 中间：<关键步骤或决策>
- 右侧：<结果/分支/总结>
- 顶部：<总标题>
- 底部：<提示/注意事项>

【信息层级】
- 一级信息（必须一眼看到）：
  1. <要点 1>
  2. <要点 2>
- 二级信息（辅助理解）：
  1. <要点 3>
  2. <要点 4>

【文字与标注】
- 图中必须出现的标签：
  - <标签 A>
  - <标签 B>
  - <标签 C>
- 强调关系：
  - 用箭头连接：<A -> B -> C>
  - 用圈注突出：<关键风险/关键动作>
  - 用星标突出：<唯一最重要结论>

【颜色（少量统一）】
- 强调色 1：<颜色名>
- 强调色 2：<颜色名，可选>
- 其余内容保持简洁，不增加无关色块

【准确性约束】
- 不新增原文不存在的功能或规则
- 不删减会影响理解的关键步骤
```

## 3) 截图任务单模板

```markdown
### 截图任务 <编号>

- 对应章节：<指南中的章节标题>
- 截图目的：<这张图帮助理解什么>
- 前置条件：<账号状态/数据准备/入口页面>
- 操作步骤：
  1. <步骤 1>
  2. <步骤 2>
  3. <步骤 3>
- 截图时机：执行到“<某状态/某提示出现>”时立即截图
- 必须包含元素：<按钮/提示/状态标签>
- 避免包含：<无关区域/敏感信息>
```

## 4) 视频录制任务单模板

```markdown
### 视频演示任务 <编号>

- 演示目标：用户看完后可以 <完成某任务>
- 建议时长：<例如 45-90 秒>
- 录制脚本：
  1. 开场（5-10 秒）：说明本段要完成什么
  2. 操作段（主体）：按步骤执行并口述关键判断点
  3. 收尾（5-10 秒）：给出成功标准与下一步建议
- 必讲要点：
  - <要点 1>
  - <要点 2>
- 可省略内容：
  - <低价值重复操作>
```
